BlobStorageException
Link copied to clipboard
A{@code BlobStorageException}is thrown whenever Azure Storage successfully returns an error code that is not200-level. Users can inspect the status code and error code to determine the cause of the error response. Theexception message may also contain more detailed information depending on the type of error. The user may alsoinspect the raw HTTP response or call toString to get the full payload of the error response if present. Note thateven some expected "errors" will be thrown as a{@code BlobStorageException}. For example, some users may perform agetProperties request on an entity to determine whether it exists or not. If it does not exists, an exception will bethrown even though this may be considered an expected indication of absence in this case.
